# 開発・貢献ガイド (CONTRIBUTING)

このプロジェクトのメンテナンスや機能追加を行うためのガイドラインです。

## 開発フロー

### 1. フロントエンドの変更
フロントエンドは静的なHTML/CSS/JSで構成されています。

*   **CSS**: `assets/css/` 内のファイルを編集します。`main.css` がエントリーポイントとなり、各コンポーネントを `@import` しています。
*   **JS**: `assets/js/` 内のファイルを編集します。
    *   `main.js`: 共通機能、トップページ
    *   `article.js`: 記事詳細ページ
    *   `transitions.js`: ページ遷移 (Barba.js) とスクロール (Lenis)
*   **HTML**: `index.html` (トップページ) や `automation/templates/article.html` (記事テンプレート) を編集します。

**注意**: `posts/` ディレクトリ内のHTMLファイルは自動生成されるため、直接編集しても上書きされます。テンプレートを変更して再生成してください。

### 2. 自動化パイプラインの変更
`automation/` ディレクトリ内の各エージェントを変更します。

*   **設定変更**: `automation/config/constants.js` (定数) や `prompts.js` (AIプロンプト) を調整します。
*   **ロジック変更**: 各エージェント (`researcher`, `generator`, `publisher`) のコードを編集します。

## コード規約

*   **JavaScript**: モダンなES6+記法を使用してください。
*   **CSS**: クラス名はBEM記法を意識し、`assets/css/` 内の適切なファイルに記述してください。
*   **コメント**: 複雑なロジックや、AIプロンプトの意図などにはコメントを残してください。

## テスト・検証

### ローカルサーバーでの確認
Barba.jsなどの機能は、ローカルファイル (`file://`) ではセキュリティ制限により動作しません。
必ずローカルサーバーを立ち上げて確認してください。

```bash
# VS CodeのLive Server拡張機能などが便利です
# または
npx serve .
```

### パイプラインのテスト
変更を加えた後は、必ずテスト実行を行ってください。

```bash
# テスト用のキーワードで実行
npm run pipeline -- --keyword="テスト"
```

## 新しいライブラリの追加
`package.json` に追加し、`README.md` やこのドキュメントにも必要に応じて追記してください。
フロントエンドのライブラリは、可能な限りCDNを利用するか、軽量なものを選択してください。
