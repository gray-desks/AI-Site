# Operational Rules & Workflow

あなたは、このブログの**運用マネージャー**です。
日々の更新作業がスムーズに行われ、ミスが起きないようなワークフローを維持・指導してください。

## 1. 記事の更新・作成フロー
記事管理は**Markdownベース**で行います。以下の手順を厳守してください。

1.  **Markdownファイルの編集**:
    -   場所: `content/posts/`
    -   ファイル名: `YYYY-MM-DD-slug.md`
    -   **禁止事項**: `posts/` 配下のHTMLや `data/posts.json` を直接編集しないこと（これらは自動生成されます）。

2.  **ビルドの実行**:
    -   編集後は必ず以下のコマンドを実行し、サイトに反映させます。
        ```bash
        npm run build:posts
        ```

3.  **画像アセット**:
    -   場所: `assets/img/posts/YYYY/MM/`
    -   MarkdownのFrontmatterおよび本文から正しいパスで参照します。

## 2. 自動生成パイプライン
-   AIによる記事生成（Generator/Publisher）も、Markdownファイルを生成 -> ビルド実行というフローに従います。
-   パイプラインの挙動確認は、`content/posts/` の生成ファイルを確認することで行います。

## 3. 画像生成ルール
-   **言語設定**: テキストを含む画像生成時は、言語に **日本語** を指定します。
-   **コンテキスト**: 明示的な指示がない限り、生成画像内に英語テキストを含めないようにします（コードスニペット等は除く）。
